# AI Agent 系统架构设计文档

**日期**: 2026-03-13
**版本**: 1.0
**状态**: 设计确认

---

## 1. 概述

本文档描述了一个本地 AI Agent 系统的微服务架构设计，包含 Skills、MCP、RAG、Memory 四大核心模块。

### 1.1 核心目标

- 构建一个可扩展、模块化的 AI Agent 系统
- 支持技能驱动的自动化任务执行
- 提供文档检索增强生成 (RAG) 能力
- 实现跨会话的持久化记忆学习

### 1.2 架构风格

微服务架构，各模块独立部署、通过 API 通信。

## 3. 技术栈

| 模块 | 技术选择 |
|------|----------|
| 前端界面 | Vue 3 + TypeScript + Element Plus |
| API Gateway | Spring Boot / Spring Cloud Gateway |
| Agent Core | Java (Spring AI / LangChain4j) |
| Skills 系统 | Markdown 定义 + Java 执行器 |
| MCP Servers | Python 独立进程 (HTTP/gRPC) |
| RAG 服务 | Java (Spring AI RAG) + Qdrant |
| 嵌入模型 | bge-m3 (Python Sidecar) |
| Memory | SQLite + 语义检索 |

## 4. 模块设计

### 4.1 Skills 系统

**职责**: 定义和执行可插拔的技能/行为。

**设计要点**:
- 技能定义：Markdown 文件格式
- 触发方式：命令式 (`/skill-name`) + 自然语言路由
- 权限控制：危险操作需要用户二次确认

**技能示例结构**:
```markdown
# Skill: debug

## 触发词
- /debug
- 帮我调试
- 查找问题

## 参数
- file: 目标文件路径
- error: 错误信息

## 执行步骤
1. 读取错误日志
2. 定位问题代码
3. 分析可能原因
4. 提供修复建议

## 权限
- 需要确认：是
```

### 4.2 MCP 系统 (Model Context Protocol)

**职责**: 连接外部工具/资源的标准协议层。

**MCP Servers 列表**:

| Server | 功能 |
|--------|------|
| filesystem | 文件读写 |
| shell | 命令执行 |
| database | SQL 查询 |
| http | API 调用 |
| git | Git 操作 |

**实现方式**:
- 独立 Python 进程，通过 HTTP/gRPC 通信
- OAuth 2.0 认证

### 4.3 RAG 系统

**职责**: 检索增强生成，提供文档知识检索能力。

**服务拆分**:

```
┌─────────────────┐   ┌─────────────────┐   ┌─────────────────┐
│  索引服务        │   │  检索服务        │   │  查询服务        │
│                 │   │                 │   │                 │
│ • 文档上传      │   │ • 相似度搜索    │   │ • 查询改写      │
│ • 分块处理      │   │ • 过滤查询      │   │ • 重排序        │
│ • 嵌入生成      │   │ • 结果聚合      │   │ • 上下文构建    │
│ • 向量存储      │   │                 │   │ • 引用追踪      │
└─────────────────┘ └─────────────────┘ └─────────────────┘
```

**技术选择**:
- 向量数据库：Qdrant
- 嵌入模型：bge-m3 (Python Sidecar)
- 优先级：检索服务优先实现

### 4.4 Memory 系统

**职责**: 跨会话持久化记忆，学习用户偏好和项目知识。

**设计要点**:
- 存储：SQLite
- 记忆类型：用户偏好
- 检索方式：语义检索 (复用 RAG 向量索引)
- 遗忘策略：时间衰减

## 5. 数据流

### 5.1 典型请求流程

```
用户请求 → API Gateway → Agent Core
                            │
                            ├──→ 检查 Memory (用户偏好)
                            ├──→ 路由到 Skill
                            ├──→ 调用 MCP 工具
                            └──→ RAG 检索上下文
                                 │
                                 └──→ 构建 Prompt → LLM → 响应
```

### 5.2 RAG 索引流程

```
文档上传 → 分块处理 → Python Sidecar 嵌入 → Qdrant 存储
```

### 5.3 RAG 检索流程

```
用户查询 → 查询改写 → Python Sidecar 嵌入 → Qdrant 相似度搜索 → 重排序 → 上下文构建
```

## 6. 接口设计

### 6.1 API Gateway 端点

| 端点 | 方法 | 描述 |
|------|------|------|
| `/api/chat` | POST | 对话请求 |
| `/api/skills` | GET | 获取技能列表 |
| `/api/skills/{name}/execute` | POST | 执行技能 |
| `/api/rag/search` | POST | RAG 检索 |
| `/api/rag/index` | POST | 文档索引 |
| `/api/memory` | GET | 获取记忆 |
| `/api/mcp/{server}` | POST | MCP 工具调用 |
